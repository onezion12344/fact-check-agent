## 结构复盘

### 布局逻辑

```
深色 Header 卡片（整体评级）
  → 统计行（4色卡：✅/⚠️/❌/总数 + 论文数/搜索轮次）
  → Section: SIFT 源码对照表
  → Section: CoVe+FIRE 独立验证表
  → Section: FABLE 影响分析网格
  → Section: Truth Sandwich 修正框
  → 方法论底部（灰色小字）
```

### 关键设计决策

1. **Stats row 放在顶部** — 用户第一眼看到整体评分，再决定是否展开看详情
2. **轮次不折叠** — 所有 section 默认展开，方便截图/分享。HTML 在文件系统中是可滚动的，没有 Telegram 截断问题
3. **绿色为主色调** — 报告主要内容正确时使用绿色系（而非中立灰色），视觉传达"可信"
4. **红色仅用于纠正框** — 只在 Truth Sandwich 修正时使用红色背景，强调定位
5. **每个主张带原文引用** — SIFT 表右侧列直接粘贴原文句子，不是 paraphrase

### 适用场景

- 用户明确要求 "事实核查 网站呈现"（本 session）"cf 给我" 时
- 核查结果需要分享给第三方查看时
- 报告非常长（40+主张，6+修正）时，纯文本会严重截断

### 不适用场景

- 用户只说了 "事实核查" 没加 "网站呈现" → 先发内联浓缩版，不主动生成 HTML
- 核查错误很少（1-2条）→ 走"修正如短"策略，不需要 HTML
- 用户正在手机上快速聊天 → HTML 需要保存到文件+打开，不适合移动即时对话
