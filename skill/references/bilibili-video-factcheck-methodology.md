# Bilibili/视频内容事实核查方法论

> 适用于用户分享B站/YouTube等视频链接，要求事实核查的场景。
> 与 logic-theory-check 配合使用（事实核查→逻辑分析）。

## 工作流

### Step 0 — 获取完整字幕/文本

视频内容与文字内容不同，需要先提取文本才能进行事实核查：

**方案A：字幕下载站（推荐）**
1. 用 pdsub.com 等字幕站查视频的 BV 号或标题
2. 如果找到字幕文件 → 用 browser_console 读取 textarea 内容获取完整 SRT
3. 如果 pdsub 没有 → 用 open-subtitles 等站

**方案B：B站页面描述 + 评论**
1. 浏览器导航到 B 站视频页
2. 滚动获取描述区和评论区内容
3. 描述区常包含事件日期、案例描述

**方案C：Mimo 视频理解**
1. mcp_mimo_mcp_mimo_video_understand — 直接理解视频内容
2. 注意：B站分享链接（b23.tv）可能被反爬拦截，需先解析出 BV 号用直链

### Step 1 — 提取事实主张

从字幕中提取所有**可验证的断言**：

- 事件（时间、地点、人物、经过）
- 数据（统计数字、比例、研究结论）
- 引用（理论名称、著作名称、名人名言）
- 法规/标准（法律条文号、标准编号）

分类标记输出，每条包含：
```
来源时间轴 | 主张内容 | 类别（事件/数据/法规/引用） | 置信度初始判断
```

### Step 2 — 并行分配核查（关键：利用 delegate_task 并发）

**并发策略**：将主张分 2-3 组并行核查（受 max_concurrent_children 限制，默认为 3）

**分组原则**：按类别分，而非按数量均分
```
批次1：历史事件（通常搜索效率高，返回快）
批次2：数据/统计（可能需要学术库）
批次3：法规/引用（可能需要政府/百科站）
```

**每个子 Agent 的 task context 必须包含**：
```
1. 明确说"你是 fact-check 子 Agent，只做事实核查"
2. 列出该组所有主张（逐条）
3. 要求每条返回：✅/⚠️/❓/❌ + 修正值 + 来源
4. 要求≥2个独立来源（新闻事件多平台交叉）
5. 要求标注"来源追溯路径"（你怎么找到这个来源的）
```

**⚠️ 已在该 session 中验证的要点**：
- 第三方转述（如"世界厕所组织说"）可能张冠李戴 → 一定要追溯原始出处，不要止步于中间引用
- 社会新闻事件通常被多家媒体转载 → 找到最早/最权威的报道
- B站UP主引用的案例可能被扭曲：找原始新闻而非UP主的描述

### Step 3 — 汇总核查结果

收到所有子 Agent 结果后，合并为一张总表：

| 序号 | 主张 | 视频声称 | 核查结果(✅/⚠️/❌/❓) | 修正值 | 来源 |
|-----|-------|---------|---------------------|--------|------|
| 1   | ...   | ...     | ✅                   | ...    | ...  |

**重要**：统计准确率，标注 what % of claims checked out.

### Step 4 — 逻辑分析（配合 logic-theory-check）

事实核查完成后，单独加载 logic-theory-check 分析论证结构：
- 识别谬误（稻草人、虚假二分、滑坡等）
- 检查推理链条是否成立
- 分析案例选择的代表性

### Step 5 — 输出综合报告

```
## 一、事实核查总表

✅ 准确项（N/10）
⚠️ 部分准确（N/10）
❌ 有问题的声明（N/10）

## 二、逻辑结构分析

论证大纲（还原视频的核心推理链条）
关键逻辑漏洞（编号逐一分析）
✅ 视频合理之处（公正性，承认对方有价值的点）

## 三、一句话结论
```

## 已知陷阱

- **B站短链接（b23.tv）**：Mimo 视频理解可能被 412 反爬拦截，需先用 BV 号转直链或直接浏览器访问
- **AI 生成的 B 站字幕可能不准**：pdsub 的字幕是用户上传，可能有 OCR 错字。逐条字幕可能有时间偏移
- **UP主描述的案例可能加入主观修饰**：如"网友说→被骂→被网暴"等，第三方报道可能不同
- **时间点的精确性**：B站UP主说"2023年"但可能是3月或5月，需要精确确认
- **案例的可验证性**：有些案例可能是道听途说（"网上说的"），实际找不到独立来源
- **捏造/歪曲案例**：UP主可能为论证需要引用不存在的案例或歪曲已有事件
